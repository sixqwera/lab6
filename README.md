# Чарушников Никита Бас-2 Лабораторная №6

# Задание 1

## Задача 1

### Текст задачи

Разработать класс с тремя целочисленными полями. Создать конструктор копирования.
Разработать метод, вычисляющий максимальную из последних цифр полей. Перегрузить метод
ToString() для формирования строки из полей класса. Реализовать дочерний класс (его
содержание предложить самостоятельно и описать в решении: какой содержательный смысл
имеют поля; реализовать конструкторы; предложить и реализовать 2-3 метода). Протестировать
все конструкторы и другие методы базового и дочернего классов.

### Алгоритм решения

Пользователь запускает программу. Программа выводит заголовок о начале работы с первым классом. Пользователь вводит первое целое число, затем второе, затем третье. При вводе букв вместо чисел программа сообщает об ошибке и просит ввести число заново. После успешного ввода всех трёх чисел программа создаёт объект первого класса и выводит его содержимое на экран. Затем программа находит максимальную последнюю цифру среди введённых чисел и выводит результат. Далее программа сообщает о проверке копирования, создаёт копию созданного объекта и выводит её данные. После этого программа выводит заголовок о начале работы со вторым классом. Пользователь вводит четыре целых числа. Программа создаёт объект второго класса, вычисляет сумму всех четырёх чисел и выводит результат, затем вычисляет произведение всех четырёх чисел и выводит результат. Программа завершает работу.

# Задание 2

## Задача 1

### Текст задачи

LineSegment double x, double y
(координаты начала и конца
отрезка на координатной
прямой)
Определить, попадает ли заданное число
в отрезок. Результат должен быть типа
bool.

### Алгоритм решения

Я запускаю программу. Сначала она настраивает отрезок: просит меня ввести координату x — я ввожу число, например 3. Если я ошибусь и введу букву, программа попросит повторить ввод, пока я не введу число. Затем программа просит ввести координату y — я ввожу 8. Программа создает отрезок с этими точками и выводит его на экран: "Создан Отрезок: [3; 8]".

После этого программа переходит к проверке операций. Она показывает длину отрезка — вычисляет её через оператор ! и выводит "Длина (!): 5". Затем программа автоматически применяет к отрезку оператор ++ — увеличивает оба конца на 1, и я вижу "После ++: Отрезок: [4; 9]".

Далее программа просит меня ввести число для проверки в диапазоне от -100 до 100 — я ввожу, например, 5. Программа через оператор < проверяет, входит ли это число в отрезок (смотрит, находится ли число между минимальным и максимальным концом отрезка включительно). Если входит — выводит "Число 5 входит", если нет — "Число 5 НЕ входит".

В конце программа пишет "Нажмите любую клавишу..." — я нажимаю любую клавишу, и программа закрывается.
